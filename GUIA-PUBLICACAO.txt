# 🚀 Guia Completo de Publicação — Gestão Empresarial
# Firebase + Vercel (do zero ao ar)

==========================================================
ANTES DE COMEÇAR — O QUE VOCÊ VAI PRECISAR
==========================================================

✅ Node.js instalado no seu computador
✅ Uma conta Google (para o Firebase)
✅ Uma conta no GitHub (github.com — crie grátis se não tiver)
✅ Uma conta na Vercel (vercel.com — crie grátis se não tiver)
✅ O projeto baixado e descompactado

→ Para verificar se o Node.js está instalado, abra o terminal e digite:
     node -v

  Se aparecer um número (ex: v20.11.0), está ok.
  Se aparecer erro, baixe em: https://nodejs.org  (clique em "LTS")
  Instale e reinicie o terminal.

==========================================================
ETAPA 1 — CONFIGURAR O FIREBASE (banco de dados na nuvem)
==========================================================

1.1 — Criar o projeto no Firebase
----------------------------------
1. Abra o navegador e acesse: https://console.firebase.google.com
2. Faça login com sua conta Google
3. Clique no botão verde "Criar um projeto" (ou "Add project")
4. No campo de nome, digite:  gestao-empresarial
5. Clique em "Continuar"
6. Na tela do Google Analytics, DESATIVE a chave (não precisa)
7. Clique em "Criar projeto"
8. Aguarde alguns segundos até aparecer "Seu projeto está pronto"
9. Clique em "Continuar"

1.2 — Ativar o banco de dados Firestore
-----------------------------------------
1. No menu esquerdo, clique em "Criação" (ou "Build") → "Firestore Database"
2. Clique no botão "Criar banco de dados"
3. Na tela que aparece, selecione "Iniciar no modo de teste"
   (isso permite ler e gravar dados sem restrições por 30 dias)
4. Clique em "Avançar"
5. Em "Local do Cloud Firestore", selecione:
     southamerica-east1  (servidor no Brasil — mais rápido para você)
6. Clique em "Ativar"
7. Aguarde o banco ser criado (pode levar 1 minuto)

1.3 — Registrar o app Web e pegar as chaves
---------------------------------------------
1. Clique no ícone de engrenagem ⚙️ no canto superior esquerdo
2. Clique em "Configurações do projeto"
3. Role a página para baixo até encontrar "Seus apps"
4. Clique no ícone  </>  (Web — parece um símbolo de código)
5. Em "Apelido do app", digite:  gestao-web
6. NÃO marque a opção Firebase Hosting
7. Clique em "Registrar app"
8. Vai aparecer um bloco de código parecido com este:

   const firebaseConfig = {
     apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXX",
     authDomain: "gestao-empresarial.firebaseapp.com",
     projectId: "gestao-empresarial",
     storageBucket: "gestao-empresarial.appspot.com",
     messagingSenderId: "123456789012",
     appId: "1:123456789012:web:abcdef1234567890"
   };

9. ⚠️ IMPORTANTE: Deixe esta tela aberta (ou copie esses valores em um bloco
   de notas). Você vai precisar deles na Etapa 2.
10. Clique em "Continuar no console"

==========================================================
ETAPA 2 — PREPARAR O PROJETO NO SEU COMPUTADOR
==========================================================

2.1 — Descompactar e abrir a pasta do projeto
-----------------------------------------------
1. Baixe o arquivo  gestao-empresarial.zip  (que o Claude te enviou)
2. Descompacte em uma pasta fácil de achar, por exemplo:
     Windows: C:\projetos\gestao-empresarial
     Mac/Linux: /Users/seunome/projetos/gestao-empresarial

2.2 — Abrir o terminal na pasta do projeto
--------------------------------------------
Windows:
  1. Abra a pasta descompactada no Explorer
  2. Clique na barra de endereço (onde aparece o caminho da pasta)
  3. Digite  cmd  e pressione Enter
  → Um terminal abre já na pasta certa

Mac:
  1. Abra o Terminal (Spotlight: Cmd+Espaço → digite "Terminal")
  2. Digite o comando abaixo (substitua pelo caminho real):
     cd /Users/seunome/projetos/gestao-empresarial

Linux:
  1. Clique com botão direito na pasta → "Abrir no terminal"

2.3 — Criar o arquivo .env com suas chaves do Firebase
---------------------------------------------------------
Ainda no terminal, copie e execute o comando abaixo UMA LINHA DE CADA VEZ.
Substitua cada valor pelo que apareceu na tela do Firebase (Etapa 1.3):

Windows (cmd):
  copy .env.example .env

Mac/Linux:
  cp .env.example .env

Agora abra o arquivo .env com um editor de texto simples:
  Windows: clique duas vezes no arquivo .env → abre no Bloco de Notas
  Mac: clique com botão direito → Abrir com → TextEdit
  (se não aparecer, o arquivo está oculto — veja dica abaixo)

⚠️ Dica Windows — ver arquivos ocultos:
  No Explorer → Exibir → marque "Itens ocultos"

O arquivo .env vai ter este conteúdo:
  VITE_FIREBASE_API_KEY=sua_api_key_aqui
  VITE_FIREBASE_AUTH_DOMAIN=seu-projeto.firebaseapp.com
  VITE_FIREBASE_PROJECT_ID=seu-projeto-id
  VITE_FIREBASE_STORAGE_BUCKET=seu-projeto.appspot.com
  VITE_FIREBASE_MESSAGING_SENDER_ID=123456789
  VITE_FIREBASE_APP_ID=1:123456789:web:abcdef123456

Substitua cada valor depois do = pelos valores do seu firebaseConfig.
Exemplo final (com dados fictícios):
  VITE_FIREBASE_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXX
  VITE_FIREBASE_AUTH_DOMAIN=gestao-empresarial.firebaseapp.com
  VITE_FIREBASE_PROJECT_ID=gestao-empresarial
  VITE_FIREBASE_STORAGE_BUCKET=gestao-empresarial.appspot.com
  VITE_FIREBASE_MESSAGING_SENDER_ID=123456789012
  VITE_FIREBASE_APP_ID=1:123456789012:web:abcdef1234567890

Salve o arquivo (Ctrl+S).

2.4 — Instalar as dependências do projeto
-------------------------------------------
No terminal (ainda na pasta do projeto), execute:
  npm install

Aguarde. Vai baixar os pacotes necessários (pode levar 1-2 minutos).
Ao final aparece algo como: "added 150 packages".

2.5 — Testar localmente (opcional mas recomendado)
---------------------------------------------------
Execute:
  npm run dev

Vai aparecer:
  Local:   http://localhost:5173/

Abra esse endereço no navegador. O app deve funcionar.
Para parar o servidor: pressione  Ctrl + C  no terminal.

ETAPA 3 — SUBIR O CÓDIGO PARA O GITHUB
==========================================================

3.1 — Instalar o Git (se não tiver)
--------------------------------------
Verifique se já tem:
  git --version

Se aparecer um número (ex: git version 2.43.0), pule para o passo 3.2.
Se aparecer erro, baixe em: https://git-scm.com/downloads
Instale com as opções padrão e reinicie o terminal.

3.2 — Criar um repositório no GitHub
---------------------------------------
1. Acesse https://github.com e faça login
2. Clique no botão "+" no canto superior direito → "New repository"
3. Em "Repository name", digite:  gestao-empresarial
4. Deixe como "Private" (privado — só você acessa)
5. NÃO marque nenhuma opção extra (sem README, sem .gitignore)
6. Clique em "Create repository"
7. Vai abrir uma página com instruções. Deixe essa aba aberta.

3.3 — Enviar o projeto para o GitHub
---------------------------------------
No terminal (na pasta do projeto), execute os comandos abaixo
UM POR UM, aguardando cada um terminar:

  git init

  git add .

  git commit -m "primeiro commit"

  git branch -M main

Agora copie o comando que aparece na página do GitHub na seção
"…or push an existing repository from the command line".
Vai ser algo assim (com seu usuário):

  git remote add origin https://github.com/SEU-USUARIO/gestao-empresarial.git

Execute esse comando copiado do GitHub. Depois execute:

  git push -u origin main

O terminal vai pedir seu usuário e senha do GitHub.
⚠️ A senha não é a senha da sua conta — é um TOKEN.

Como gerar o token:
  1. No GitHub, clique na sua foto → Settings
  2. Role até o final → "Developer settings"
  3. "Personal access tokens" → "Tokens (classic)"
  4. "Generate new token" → "Generate new token (classic)"
  5. Em "Note", escreva: gestao-deploy
  6. Em "Expiration", selecione: 90 days (ou No expiration)
  7. Marque a opção: ✅ repo
  8. Clique em "Generate token"
  9. COPIE o token que aparece (começa com ghp_...)
     ⚠️ Ele só aparece uma vez — guarde num lugar seguro!

Volte ao terminal:
  - Usuário: seu e-mail ou nome de usuário do GitHub
  - Senha: cole o token (ghp_...)

Ao final aparece: "Branch 'main' set up to track remote branch 'main'"
Seu código está no GitHub! ✅

ETAPA 4 — PUBLICAR NA VERCEL
==========================================================

4.1 — Criar conta na Vercel
------------------------------
1. Acesse https://vercel.com
2. Clique em "Sign Up"
3. Selecione "Continue with GitHub"
4. Autorize a Vercel a acessar seu GitHub

4.2 — Importar o projeto
--------------------------
1. No painel da Vercel, clique em "Add New..." → "Project"
2. Na lista de repositórios, encontre "gestao-empresarial"
3. Clique em "Import"
4. Em "Framework Preset" deve aparecer automaticamente: Vite ✅
5. Não altere nada em "Build and Output Settings"

4.3 — Adicionar as variáveis de ambiente
------------------------------------------
⚠️ Esta é a etapa mais importante — sem isso o app não conecta ao Firebase!

Ainda na tela de import da Vercel, clique em "Environment Variables"
(fica logo abaixo das configurações de build).

Adicione cada variável clicando no campo, digitando o nome e o valor:

Variável 1:
  Nome:  VITE_FIREBASE_API_KEY
  Valor: (cole o apiKey do seu firebaseConfig)

Variável 2:
  Nome:  VITE_FIREBASE_AUTH_DOMAIN
  Valor: (cole o authDomain)

Variável 3:
  Nome:  VITE_FIREBASE_PROJECT_ID
  Valor: (cole o projectId)

Variável 4:
  Nome:  VITE_FIREBASE_STORAGE_BUCKET
  Valor: (cole o storageBucket)

Variável 5:
  Nome:  VITE_FIREBASE_MESSAGING_SENDER_ID
  Valor: (cole o messagingSenderId)

Variável 6:
  Nome:  VITE_FIREBASE_APP_ID
  Valor: (cole o appId)

Após adicionar as 6 variáveis, clique em "Deploy".

4.4 — Aguardar o deploy
-------------------------
O deploy leva cerca de 1-2 minutos.
Quando terminar, vai aparecer uma tela de confetes 🎉 e um link como:
  https://gestao-empresarial-abc123.vercel.app

Clique no link — seu app está no ar e funciona em qualquer dispositivo! ✅

==========================================================
ETAPA 5 — VERIFICAR SE ESTÁ TUDO FUNCIONANDO
==========================================================

1. Abra o link da Vercel no celular e no computador ao mesmo tempo
2. Cadastre um produto no computador
3. Atualize a página no celular — o produto deve aparecer automaticamente

Se o produto apareceu: está tudo funcionando com sincronização em tempo real! ✅

Se aparecer erro no console do navegador (F12 → Console), verifique:
  - Se todas as 6 variáveis de ambiente foram adicionadas na Vercel
  - Se copiou os valores corretamente (sem espaços extras)
  - Se o Firestore foi ativado no Firebase (Etapa 1.2)

==========================================================
ETAPA 6 — ATUALIZAÇÕES FUTURAS (quando o app for melhorado)
==========================================================

Toda vez que receber uma versão atualizada do app, é só:

1. Substituir os arquivos na pasta do projeto
2. Abrir o terminal na pasta e executar:

   git add .
   git commit -m "atualização do app"
   git push

3. A Vercel detecta a mudança e atualiza automaticamente em ~1 minuto.
   Sem precisar fazer nada na Vercel! 🔄

==========================================================
❓ PROBLEMAS COMUNS
==========================================================

Problema: "npm não é reconhecido como comando"
Solução: Instale o Node.js em https://nodejs.org e reinicie o terminal

Problema: "git não é reconhecido como comando"
Solução: Instale o Git em https://git-scm.com/downloads e reinicie o terminal

Problema: App abre mas não salva dados
Solução: Verifique as 6 variáveis de ambiente na Vercel:
  Vercel → seu projeto → Settings → Environment Variables

Problema: Erro "permission denied" no git push
Solução: Gere um novo token no GitHub (passo 3.3) e use como senha

Problema: A Vercel não encontra o repositório
Solução: Na Vercel → Settings → Git → Connect → autorize o repositório
